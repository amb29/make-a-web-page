FUNDAMENTALS OF THE WEB

LESSON 1: THE BASICS OF THE WEB AND HTML
Introduction

There are at least five important subjetcs to be studied as basic elements on the web: the web itself, the code language HTML, the repository URL, the protocol HTTP and proper web aplications. First of all the web - or world wide web.

The World Wide Web

The World Wide Web is a collection of HTML documents, which stands for Hypertext Markup Language. The documents are all linked to one another throught Hyper Links. All kinds of file can exist on the web, music, video, images - HTML documents can refer to anything.

There are four main actors in the web: users, computers + browsers, the internet itself and servers. The user put up an internet adress on his browser and it makes a request via the internet to the servers, using the HTTP protocol. The servers respond with files displaced by the browser. Servers are computers which host files, and the HTTP is the main protocol used on internet communications. Web documents, as said before, are written in a language called HTML, the next topic.

HTML - Hyper Text Markup Language

HTML or Hyper Text Markup Language is the language used to write documents placed on the web. It is made of four components: text content, or the text that can be seen; Markup, or what the text looks like; References to other documents; and Links to other pages. Text content is plain text, while markup is used to mark texts up.

HTML Markup

Texts are marked up using tags. Usually an element is made of an opening tab, a closing tab and contents, for example:

Ex: <name>contents</name> ----- This is called an element

In an element the contents inside the opening and closing tags are marked up with the effects of that tag. There are several kinds of tags, which one with its own markup effect on text, like the <b> bold tag, <em> emphasis - italics tag, Anchor tags <a>, used to make links, image tags <img>, among others. There are also tags with attributes <tag attr = "value">, which have values. It is important to say that image tags are void tags, that don´t need a closing tab.

Whitespace

All that is put in HTML turn in a single space. If a user writes a text in multiple lines, it appears in a single line, in whitespace. It is necessary to force multiple lines using tags. The break tag <br> is the first way. It is a void tag to be used after any line the users wants a break. The other way is with paragrarh tag <p>, which is not a void tag. The main difference between those two tags is that <br> is an inline tag, while <p> is a block one.

Inline and Block elements

Block and inline tags have different behaviour. Inline elements fit within a line, somentimes ending it, like <br> tag. Block tags make and an invisible box around the text, that can have height and width. Examples of inline elements are: <a>, <span>, <br>, <img>, <strong>. Some examples of block elements: <div>,<p>, <form>.

Container elements

Containers are elements that contain stuff inside them. There are two container tags: <span> and <div>. Span is an inline element, while div is a block one. They both can use CSS, which is a language to give styles to documents.

Computers and stupidity

It is good to remember that computers are stupid, and that is not a bad thing to say. They are stupid because they only interpret instructions literaly.

LESSON 2: CREATING A STRUCTURED DOCUMENT WITH HTML
HTML, CSS and Javascript

HTML files are the structure of a webpage, CSS files are the style of a page, and Javascript files are the interactive components of a page. These languages define the rules for how webpages behave.

HTML, Structure and CSS: tree-like structure

Webpages have branched structures like trees (tree-like structure). Branches are formed in coding, from big elements to small ones. The same structure may result in different looking webpages, depending on the style imprinted by CSS. It is important to remember that HTML and CSS are both languages. HTML controls the structure of a webpage, while CSS controls the style of a page.

HTML, CSS and the DOM

The DOM is the tree-like structure of a page. HTML is the language of the web. It has a specific syntax and rules. The basic word in the language of HTML is tag. The browser turns tags into elements, forming trees. It knows how to do it because of the DOM

The DOM or Document object module is a form to represent and interact with elements in HTML. The tree-like structure observed in developers tools section of the browser shows a DOM tree that the browser builds from a HTML document.

Each HTML tag creates an element in the DOM that browsers use to display the page. The content of a tag can be text, empty or even another tag. Every tag has a type or name that defines the element that will be created. Also tags can have attributes with values. With tags HTML documents define the structure and content of a page.

To define the style of a page it is necessary to use another language: CSS. CSS allows the use of specific syntax and rules to change how elements look on the page (fonts size, color, background, orders, positions, etc.)
Ex: It is possible with CSS to state that all <h1> elements that belong to the class "article" must be dark blue.

To tie structure and style together the HTML document can provide the browser with information on what style to use and where to get it.

Specific Rules of how elements work

Once the language is known, it is possible to look to the HTML code and get a good understanding of how the page organize and looks like, or, on the other way, how to break down a webpage into code.

Everything in a HTML document is retangular - a box. There are other shapes in the world, like circles and triangles, that can be used in code with the help pf CSS styles. It is importante to focus that in a webpage everything is a box! Different shapes are made with style

To use HTML one has to think in terms of boxes, asking: how do i draw a box in every element? How do i do it to reach simple elements, like an image or a text?

Boxify a design

The design of the page must be put into boxes, to facilitate the HTML and CSS written work. Any method works to boxify: doodle in paper, print the page, or with ones imagination - the most efficient way must be chosen. A new design must be approached from big boxes to small boxes. Bigger boxes can be filled later with smaller ones.

The <div> tag can be used to define boxes in code.
ex: <div>my apps</div>

Styles on boxes

To put style on boxes it is necessary to use CSS. For each element a class attribute must be given. A class attribute is like a label on a box, something made to remember its content. Boxes can have several labels, as tags, that can belong to multiple classes. It is good to have class names that make sense, that can be understood just by looking at the code.
ex: <div class="tittle'">my apps</div>

Text editors (for programming)

Programming language can be written in any editor, but there are some specific tools for doing it, like sublime editor. When writing code, it is highly recommended the use of a programming text editor. It makes the life of a programmer easier.

LESSON 3: APPLYING STYLES TO HTML WITH CSS
CSS

CSS stands for Cascading Style Sheets. It is a computer language that tells html files what they should look like. It controls the style of a html document. Cascading means that the most specific rule is applied to every element.

CSS uses selectors to define particulars styles to be applied in each element.

An important CSS concept is inheritance, which requires abstract thinking. This concept relies on ancestor-descendent relationship to operate, that can be seen on document trees of an html documents. By this mechanism properties are applied to a specific element and its descendents, just like in biology.

Avoidance of repetition

In programming it is very important to avoid repetition. A repetition in code may lead to more work, since a little change in something means that all repeated code must be changed also. This may cause trouble and confusion. CSS helps programmers to avoid repetition. Using <div> or other elements it is possible to control all changes in the document appearence with the CSS code, making the job easier and faster.
